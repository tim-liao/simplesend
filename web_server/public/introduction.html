操作參考 
import { sendemailwithattachment, sendonlyemail } from "simplesend";

const body = {
user_id: 0, //your user ID //required
nameFrom: "example.com", // your verify name //required
emailTo: "email@example.com", //required
emailBcc: "email@example.com",
emailCc: "email@example.com",
emailReplyTo: "email@example.com",
emailSubject: `example`, //required
emailBodyType: "text", //required //'html' or 'text'
emailBodyContent: `Hello world!`, //required
trackingOpen: "yes", //required  //'yes' or 'no'
trackingClick: "yes", //required  //'yes' or 'no'
trackingLink: "https://example.com", //if you want to track your click ,you have to provide the link you want to track in your html content
};
const apiKey = "your api key it will provide from our web service";
const filePath = "your/file/path";
//send email with attachment
sendemailwithattachment(body, filePath, apiKey);
                  
// without attachment
sendonlyemail(body, apiKey);

為何要使用 SIMPLE SEND ？

我們的網站之所以可以流暢，是依靠這些服務達成...

EC2

我們主要運用 AWS 提供的 EC2 服務，將 WEB SERVER 架設在雲端電腦上，讓伺服器不架設在自己的電腦。

SES

我們透過 AWS 的 SES(Simple Email Service) 來進行郵件投遞的工作。

S3

為了讓使用者可以寄送附帶檔案的郵件，我們運用了 AWS 提供的 S3 服務，將使用者提供的附加檔案儲存至這個雲端儲存空間，不會佔用過多 WEB SERVER 所在的雲端電腦的空間，提高 WEB SERVER 的可用率。

SNS

我們運用 AWS 提供的 SNS(Simple Notification Service) 服務，將寄件後的郵件是否真正投遞到收件者的郵箱狀態進行資訊回傳，讓你的郵件寄出後不會不知去向。

AWS

除了上面這些，我們還使用了其他的AWS服務來達到我們的需求，例如： RDS (作為資料庫儲存使用者的資料)、CloudWatch（儲存完整的SNS回傳資料）。

RabbitMQ

其他部分我們還運用 RabbitMQ 將接收到的寄件需求交由另外一個 WORKER 進行實際寄件工作，讓使用者將寄件需求提交後可以快速提交新的寄件需求給我們的網站，不將壓力囤積在使用者身上！
